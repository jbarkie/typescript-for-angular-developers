# Software Catalog — Teaching App

The working application for the TypeScript-for-Angular course. Built on Angular 21.2 with signals, `httpResource`, Tailwind + DaisyUI, Mock Service Worker + **msw-lens**, Sheriff, and Zod.

This is the place where the TypeScript lessons from the koans land in real code.

## Scenario switching with msw-lens

The mocked backend is not just one set of responses — it's five. Switch between them with a single CLI command:

```bash
npm run lens
```

This launches an interactive picker. Choose a scenario (for example `empty`, `server-error`, `slow`, `malformed-data`) and the file watcher rewrites `src/mocks/active-scenarios.ts`. Vite HMR picks it up. Flip scenarios mid-demo to show every branch of a discriminated-union state type, or to light up the Zod-at-the-boundary story on Day 3.

Scenarios are declared in the YAML manifests next to the handlers:

- `src/mocks/catalog/catalog.yaml` — `GET /api/catalog` scenarios
- `src/mocks/catalog/catalog-post.yaml` — `POST /api/catalog` scenarios

The active scenario for each endpoint lives in `src/mocks/active-scenarios.ts` — **generated, don't hand-edit except to bootstrap**.

## Setup

```bash
npm install
npm start
```

Open http://localhost:4200. Mock Service Worker runs automatically — the app fetches a seeded catalog from `/api/catalog`. No real backend.

## The shape

```
src/
  app/
    areas/
      catalog/              ← the one feature area
        types.ts            ← the discriminated union + derived payload types
        list.ts             ← list page (Day 2 PM refactor target)
        add.ts              ← add page (Day 3 PM refactor target)
        catalog.routes.ts
    shared/
      nav-config.ts         ← `as const satisfies` in real use
    app.ts                  ← shell / nav
    app.config.ts
    app.routes.ts
  mocks/
    browser.ts              ← setupWorker entry, aggregates handlers
    active-scenarios.ts     ← generated by `npm run lens`
    catalog/
      handlers.ts           ← branches on activeScenarios
      catalog.yaml          ← GET /api/catalog manifest
      catalog-post.yaml     ← POST /api/catalog manifest
```

## Not included, on purpose

- No `signalStore`. Students saw that in Applied Angular. Keeping it out so the TypeScript lessons are what they notice.
- No auth, guards, or feature flags.
- No per-component OnPush. Default in Angular 21.

## Gotchas

- MSW runs in dev. The `start()` call is in `main.ts`, before Angular bootstraps. The service worker lives at `/mockServiceWorker.js` — served from `public/`, handled by the Angular build.
- Tailwind is v4, configured via `.postcssrc.json`. DaisyUI is loaded through `@plugin 'daisyui'` in `src/styles.css`. No `tailwind.config.js` needed.
- Sheriff is installed but not wired to ESLint. Config is at `sheriff.config.ts` and available for reference. When the course covers Sheriff enforcement, add the ESLint plugin then.
